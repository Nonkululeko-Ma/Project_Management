**Kanban Board Customization**
**Overview**
This Kanban board is set up to track project tasks and make work more organized.

**Custom Columns**
To Do – Tasks that need to be started.
In Progress – Tasks that are currently being worked on.
Testing – (New) For tasks being checked for errors before completion.
Blocked – (New) For tasks stuck due to issues or missing information.
Done – Completed tasks.

**Linked Issues & Labels**
Issues are linked to match tasks with their GitHub Issues.

**Labels used**

feature → New features
bug → Problems that need fixing
enhancement → Improvements

**Task Assignments**
Tasks are assigned to team members using @mentions.
Each task has a clear description and goal.

**Kanban Board Definition**
A Kanban board is a visual tool used to manage tasks and track progress efficiently. It helps teams organize work by moving tasks through different stages, making workflow clear and structured.

**How Our Board Works**
1. Visualizing Workflow
- Our board has columns that represent different stages of work: To Do, In Progress, Testing, Blocked, and Done.
- Tasks move from left to right, showing progress at a glance.
- 
**Limiting Work-in-Progress (WIP)**

- Each column has a WIP limit to prevent too many tasks from piling up.
- This keeps the team focused and avoids delays caused by multitasking.

**Supporting Agile Principles**

- The board ensures continuous delivery by tracking work efficiently.
- It adapts to changes by allowing tasks to move flexibly based on priority.
- The Testing and Blocked columns help in identifying issues early, improving quality.

For Example:
Our Kanban board tracks tasks visually, limits WIP to prevent overload, and supports Agile by enabling continuous progress and quick adjustments.
